# Domain docs

Perfect Six uses a single repository-wide domain context.

Before domain-sensitive work, read `CONTEXT.md` when it exists and any relevant records under `docs/adr/`. Proceed silently when either location is absent.

Use the vocabulary defined by `CONTEXT.md` in issue titles, specifications, tests, and implementation. If a required concept has no established term, note the gap for domain-modeling work instead of silently inventing competing terminology.

Surface any conflict with an existing architecture decision record rather than overriding it.
